# 效果图拆解工作流

当用户上传一张效果图/截图，要求据此搭建空间界面时，按本流程把图变成"面板清单 + 跳转表"，确认后再生成代码。

## 步骤

### 1. 识别独立面板

把图中每个独立的悬浮矩形区域当作一个面板（Panel/Screen）。常见信号：
- 有独立卡片背景/圆角/阴影的区域
- 带标题栏或右上角关闭按钮（✕）的窗口
- 底部独立的工具板、便签板、控制条

为每个面板分配一个路由名（kebab/snake：`hub`、`board_a`、`board_b`…）。

### 2. 记录每个面板的内容

对每个面板记录：
- **标题**：图中显眼的标题文字
- **正文/摘要**：段落文字
- **控件**：按钮（文字）、关闭按钮、便签（颜色 + 文字）、列表项、3D 模型占位
- **是否含 3D**：图中是否有立体模型/角色/物体

### 3. 提取跳转关系

图上的箭头、连线、"继续/下一步/进入"按钮、关闭按钮都代表跳转。整理成跳转表：

| 来源面板 | 触发元素 | 目标面板 | 动作 |
|----------|----------|----------|------|
| hub | 点击任意便签 | board_a | navigate |
| board_a | "继续学习" 按钮 | board_b | navigate |
| board_a | ✕ 关闭 | hub | popBackStack |
| board_b | ✕ 关闭 | hub | popBackStack |

若用户在图上手画了编号箭头（如 1→2→3），按编号顺序推断主流程，关闭/返回单独列。

### 4. 与用户确认

把"面板清单 + 跳转表"用简短列表回给用户确认，特别是：
- 起始面板（startDestination）是哪个
- 多个"关闭"是回到上一页还是统一回起点
- 哪些面板真的要内嵌可加载的 3D 模型（需要 `.glb`/`.usdz` 资源）

确认后再按 SKILL.md 第 4–6 步生成代码。

## 注意

- 效果图常是多人协作/营销示意，**不要照搬不可实现的元素**（如真实头像、实时在线人数）。用占位文本或静态示意代替，并向用户说明哪些是示意。
- 图中文字若是特定语言，界面文案沿用该语言。
- 拿不准的区域按"最可能的单一面板"处理，并在确认环节询问。
